# Anneal proposals

Documents in this directory describe possible future mechanisms. They are not current product
commitments and must not be used as implementation-status references.

- [`input-sensing.md`](input-sensing.md) — observe reads to propose declarations and improve
  diagnostics.
- [`remote-cache.md`](remote-cache.md) — share trusted content-addressed results after local
  correctness gates are met.
- [`linux-vm.md`](linux-vm.md) — provide Linux-equivalent execution enforcement on macOS.
- [`../simplex-rules.md`](../simplex-rules.md) — first-party Simplex package and build rules;
  retained at its active working path.

Every proposal should name its prerequisites, security/correctness boundary, rollout gate, and
interaction with the current product.
